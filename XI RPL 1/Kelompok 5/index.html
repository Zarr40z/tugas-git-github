apa itu git fetch?

git fetch adalah perintah untuk mengambil update terbaru dari repo tanpa langsung menggabungkan ke branch lokal. Data yang bisa yang bisa diambil seperti :

- commit baru

- branch baru

Perintah jika hanya memiliki 1 remote:

git fetch

Perintah jika hanya ingin mengambil update dari remote yang bernama origin:

git fetch origin

Apa itu Pull Request?

Pull Request adalah fitur dari GitHub untuk mengusulkan perubahan kode dari suatu branch(fork) supaya bisa digabung ke branch utama. Pull Request biasanya digunakn dalam kolaborasi supaya kode bisa dicek dulu sebelum masuk ke repository utama

alur penggunaan Pull Request:

Misalnya kita sudah nge add, commit dan push suatu perubahan, lalu masuk ke repo di github terus bakal ada "compare & pull request. Terus klik "compare & pull request" pilih branch asal(yang di edit) ke branch tujuan, lalu berikan deskripsi dan klik Pull Request. Sesudah itu bisa direview dan diskusi dengan kelompok dan jika sudah setuju baru merge di branch tujuan.

opsional : jika branch yang sudah di merge sudah tidak diperlukan bisa dihapus

jadi kesimpulannya adalah

Git Fetch

Git fetch digunakan untuk mengambil perubahan terbaru dari remote repository tetapi tidak langsunng mengubah branch lokalnya, yang membuat lebih aman karena bisa kita cek terlebih dahulu sebelum di merge.


Pull Request (PR)

Pull Request digunakan untuk mengajukan suatu perubahan kode dari satu branch ke branch lainnya, dan dengan Pull Request kita bisa melakukan review, diskusi dan testing sebelum menggabungkan kode kode tersebut
